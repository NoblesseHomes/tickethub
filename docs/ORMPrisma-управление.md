# Prisma ORM: команды консоли с описанием и применением

Ниже — базовая шпаргалка по Prisma ORM именно в виде консольных команд: что делает каждая команда и когда её использовать. Команды подходят для типичного Node.js/TypeScript-проекта с Prisma. [web:1][web:5]

---

## 1) Установка Prisma

```bash
npm install prisma @prisma/client
```

**Что делает:** ставит сам Prisma CLI и Prisma Client.  
**Когда использовать:** когда подключаешь Prisma в новый проект или добавляешь ORM в существующий. [web:1]

---

## 2) Инициализация проекта Prisma

```bash
npx prisma init
```

**Что делает:** создаёт базовую структуру Prisma в проекте, включая файл `prisma/schema.prisma` и обычно пример переменной `DATABASE_URL` в `.env`.  
**Когда использовать:** в самом начале, когда Prisma ещё не настроена. [web:1][web:5]

---

## 3) Настройка схемы

Открой файл:

```bash
prisma/schema.prisma
```

**Что делает:** это не команда, а главный файл, где описываются модели, источник данных и генератор клиента.  
**Когда использовать:** когда нужно создать таблицы, связи, индексы, enum и другие структуры. [web:2][web:5]

Пример:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id    Int     @id @default(autoincrement())
  email String  @unique
  name  String?
}
```

---

## 4) Генерация клиента

```bash
npx prisma generate
```

**Что делает:** генерирует типизированный Prisma Client на основе схемы.  
**Когда использовать:** после изменения `schema.prisma`, чтобы код приложения видел актуальные модели и поля. [web:1][web:3]

---

## 5) Создание миграции

```bash
npx prisma migrate dev --name init
```

**Что делает:** создаёт миграцию и применяет её к локальной базе данных.  
**Когда использовать:** при первом создании схемы или после любых изменений моделей в процессе разработки. [web:1][web:5]

Пример имени миграции:

```bash
npx prisma migrate dev --name add_user_table
```

**Практическое применение:** удобно, когда ты добавил поле `name` или новую модель `Post` и хочешь зафиксировать изменения в базе. [web:1]

---

## 6) Применение миграций в production

```bash
npx prisma migrate deploy
```

**Что делает:** применяет уже созданные миграции без их генерации.  
**Когда использовать:** на сервере или в production-среде, где миграции должны накатываться из готовых файлов. [web:5]

**Практическое применение:** полезно в CI/CD или при деплое приложения.

---

## 7) Открыть Prisma Studio

```bash
npx prisma studio
```

**Что делает:** запускает веб-интерфейс для просмотра и редактирования данных в базе.  
**Когда использовать:** когда нужно быстро проверить записи, вручную исправить данные или посмотреть связи. [web:1]

**Практическое применение:** удобно для отладки и ручной проверки данных без SQL-клиента.

---

## 8) Синхронизация схемы без миграций

```bash
npx prisma db push
```

**Что делает:** отправляет схему в базу напрямую, без создания миграций.  
**Когда использовать:** для быстрого прототипирования или тестовых проектов, где история миграций не критична. [web:5]

**Практическое применение:** быстро поднять структуру БД, если не нужен полноценный migration workflow.

---

## 9) Сброс базы данных

```bash
npx prisma migrate reset
```

**Что делает:** удаляет данные, пересоздаёт базу и заново применяет миграции.  
**Когда использовать:** в разработке, когда нужно полностью очистить локальную БД и начать заново. [web:1]

**Практическое применение:** полезно, если структура сильно изменилась и проще пересоздать данные, чем чинить старое состояние.

---

## 10) Проверка состояния миграций

```bash
npx prisma migrate status
```

**Что делает:** показывает, в каком состоянии миграции относительно базы.  
**Когда использовать:** если есть подозрение, что база и миграции рассинхронизированы. [web:5]

**Практическое применение:** диагностика проблем на dev/staging сервере.

---

## 11) Интроспекция существующей базы

```bash
npx prisma db pull
```

**Что делает:** подтягивает структуру уже существующей базы в `schema.prisma`.  
**Когда использовать:** если база уже есть, а схему Prisma нужно сгенерировать по ней. [web:5]

**Практическое применение:** полезно при подключении Prisma к старому проекту с готовой БД.

---

## 12) Просмотр справки по командам

```bash
npx prisma --help
npx prisma migrate --help
npx prisma db --help
```

**Что делает:** показывает доступные команды и параметры.  
**Когда использовать:** если нужно быстро посмотреть синтаксис или ключи без поиска по документации. [web:1]

**Практическое применение:** помогает вспомнить точную команду для конкретной задачи.

---

## Частый рабочий сценарий

```bash
npm install prisma @prisma/client
npx prisma init
# редактируешь prisma/schema.prisma
npx prisma migrate dev --name init
npx prisma generate
npx prisma studio
```

**Что делает:** это стандартный путь от установки до первых таблиц и просмотра данных.  
**Когда использовать:** при старте нового проекта на Prisma. [web:1][web:5]

---

## Полезные замечания

- `migrate dev` — для разработки, `migrate deploy` — для production. [web:1][web:5]
- `db push` удобен для быстрого прототипа, но обычно не заменяет полноценные миграции. [web:5]
- После изменения схемы почти всегда нужен `prisma generate`. [web:1][web:3]
